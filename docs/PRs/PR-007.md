# PR-007 — Roster Workspace

## Purpose

Promote Roster from a placeholder into the working season-management hub that future Tryouts, Attendance, Practice, Rotations, and Live Match features can share.

## Included

- Roster becomes the primary player-management route
- Legacy `/players` route redirects to `/rosters`
- Season roster summary with Available, Limited, Injured, and Away counts
- Search by player name, jersey number, and position
- Filters for availability, starters, captains, liberos, and injuries
- Dense row-based roster list with team-aware role colors
- Shared slide-over player and roster editor
- Availability choices: Available, Limited, Injured, Away, and Inactive
- Coach Hub roster readiness summary
- Product architecture and Tryouts/Public Registration documentation

## Deferred

- One-tap row action menu
- Bulk actions and drag ordering
- Attendance history
- Match-specific availability
- Tryout implementation
- Firebase, authentication, and synchronization
