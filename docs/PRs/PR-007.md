# PR-007 — Team HQ Roster Workspace

## Purpose

Separate long-term player profile management from active coaching preparation.

- **Players** manages who the athletes are.
- **Roster / Team HQ** manages how the coach uses them.

## Included

- Restored the dedicated Players route for creating and editing player profiles
- Rebuilt Roster as an ESPN-inspired Team HQ workspace
- Added Players, Lineup, and Stats views inside Team HQ
- Added visual player cards with jersey number, position, availability, roles, and season snapshot stats
- Added a persistent player focus panel on iPad/desktop
- Added a stacked focus panel on phones
- Added starting-six court layout, libero area, and bench list
- Added persistent starter, libero, captain, and availability controls
- Added season leaderboards for kills, aces, digs, blocks, and assists
- Added next-match context from Schedule
- Updated Coach Hub links and readiness metrics
- Preserved the Organization → Team → Season context bar
- Preserved Tryouts and Public Registration planning documentation

## Current Stats Note

The statistics interface is ready for the future Live Match data pipeline. Seed players currently display representative prototype values so the layout can be evaluated; user-created players display empty values until real match-stat persistence is implemented.

- Saved lineup presets
- Match-specific rosters
- Real statistics generated from Live Match
- Player photos
