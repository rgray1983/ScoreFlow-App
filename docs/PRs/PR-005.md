# PR-005 — Players, Rosters, Context Switching, and Reusable Slide-Over Editor

## Status

Ready for local testing and review.

## Branch

`agent/pr-005-players-rosters-slide-over`

## Purpose

Introduce reusable player profiles and season-specific roster memberships while establishing shared context selection and right-side editor patterns for future Coach modules.

## Scope

- Add strict TypeScript models for players and roster memberships.
- Keep reusable player profiles separate from season roster details.
- Add local-first player and roster persistence to the existing workspace store.
- Migrate older local workspace records safely when player arrays are missing.
- Add representative player and roster seed data.
- Replace the Players placeholder with a working directory for the active organization, team, and season.
- Add player creation and editing.
- Store jersey number, position, status, captain, libero, and starter flags per season roster.
- Update the Coach Hub with the active roster count.
- Add a reusable `ContextBar` for changing organization, team, and season without leaving the current page.
- Show static context labels when only one option exists and selectors when switching is possible.
- Add a reusable `SlideOver` component.
- Use a right-side eased entrance, fading blur backdrop, and mobile full-width behavior.
- Compress the editor for phone landscape so normal forms fit without unnecessary scrolling.
- Preserve the existing scoreboard and all PR-004 program structure behavior.

## Context Interaction

The Players page now treats Organization → Team → Season as shared application context rather than navigation. Changing a selector immediately updates the active workspace, player directory, roster count, and new-player destination. A compact Manage link returns to Teams only when the user needs to create or change the underlying program structure.

When only one option exists, the bar shows its name as a compact label instead of wasting space on an unnecessary dropdown. The component is reusable for Schedule, Live Match, Rotations, Reports, Attendance, and future context-aware modules.

## Slide-Over Interaction

The editor mounts off-screen at the right edge, then enters using a smooth `cubic-bezier(.16, 1, .3, 1)` transition with a small scale settle. The backdrop fades and blurs separately. Closing reverses the transition before the component unmounts.

On phones the editor becomes full-width. In short phone landscape it becomes wider, reduces header and field height, uses four columns where practical, and hides the optional long notes field so primary player and roster data remain in the viewport.
