# PR-001 — Establish Pull Request Documentation Workflow

## Status

Ready for local testing and review.

## Branch

`agent/pr-001-docs-pr-workflow`

## Purpose

Create the permanent `docs/PRs` folder and define the ScoreFlow branch, pull request, testing, and documentation workflow.

## Scope

- Create `docs/PRs/README.md`.
- Create this initial PR record.
- Establish sequential PR numbering.
- Establish `agent/` branch naming.
- Confirm that the repository is the source of truth.
- Confirm that changes are tested locally before merging.

## Files Changed

- `docs/PRs/README.md`
- `docs/PRs/PR-001.md`

## Implementation Notes

This PR changes documentation only. It does not alter ScoreFlow application behavior, styling, Firebase logic, service-worker behavior, or build output.
