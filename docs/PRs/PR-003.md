# PR-003 — Establish ScoreFlow Coach Design System

## Status

Ready for local testing and review.

## Branch

`agent/pr-003-coach-design-system`

## Purpose

Establish the visual, responsive, and documentation foundation for ScoreFlow Coach before functional modules are built. The design follows the Coach Milestones core direction: volleyball-first, coach-focused, mobile and iPad primary, premium ScoreFlow styling, thumb-friendly controls, minimal interface clutter, and future-aware architecture.

## Scope

- Replace the generic light foundation with a dark premium sports workspace.
- Build a useful Coach Hub dashboard using representative placeholder data.
- Establish ScoreFlow red and gold action hierarchy.
- Establish reusable color, spacing, radius, panel, button, card, status, and typography tokens.
- Use a compact navigation rail for iPad and desktop.
- Use persistent bottom navigation for primary phone actions.
- Keep the application shell confined to `100dvh`.
- Restrict scrolling to the route content region instead of the full page.
- Support safe-area insets for installed iPhone and iPad PWAs.
- Add height-specific compression for short landscape viewports.
- Prevent iPad landscape performance values from being truncated.
- Add living product, architecture, UI, and feature documentation.
- Expand the roadmap to include scheduling, practice planning, tournaments, availability, attendance, calendar sync, and future integrations.
- Preserve every existing route as a clearly styled placeholder.
- Do not connect Firebase or build working team, roster, schedule, match, rotation, or reporting features yet.
- Do not modify the existing scoreboard app.

## Implementation Notes

The main application frame uses a fixed-height grid with `height: 100dvh`, `min-height: 0`, and hidden outer overflow. The top bar and navigation remain fixed within the app frame. Only `.route-viewport` scrolls when a device cannot display all dashboard content at once.

The responsive strategy starts with iPad-sized navigation and then adapts downward. At phone widths, the primary sections and a More control appear in a bottom navigation bar. Secondary destinations remain available through the slide-out menu.

At iPad landscape widths, the Last Match performance metrics switch to a two-column grid so complete values remain visible without widening the dashboard beyond the viewport.

The documentation directory is now the living source of truth for product intent, shared entities, future connections, interface standards, and implementation order. The planning rule is: before a major feature PR begins, document its effects on future systems, offline behavior, permissions, and mobile/iPad workflows.

The Coach Hub communicates the planned product direction without introducing persistent data. It includes a next-match card, quick actions, practice plan, last-match metrics, local-save status language, and links to placeholder routes.

No external icon package was added. The temporary symbols are lightweight placeholders so the design system can be tested before permanent ScoreFlow icons are selected.

## Known Limitations

- Dashboard data is representative placeholder content only.
- Navigation symbols are temporary and will be replaced by the final icon set.
- Firebase, authentication, offline data storage, and scoreboard integration are not included.
- Dashboard cards are not yet customizable.
- Detailed accessibility testing with screen readers remains a later validation step.
