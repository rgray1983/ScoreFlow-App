# PR-002 — Create ScoreFlow Coach App Foundation

## Status

Ready for local testing and review.

## Branch

`agent/pr-002-scoreflow-coach-foundation`

## Purpose

Create a separate `ScoreFlow-Coach` application home for coach-facing features without changing the existing ScoreFlow scoreboard app in the repository root.

## Scope

- Create the dedicated `ScoreFlow-Coach` folder.
- Add a responsive Coach dashboard shell.
- Add placeholder navigation for Home, Teams, Players, Rosters, Live Match, Rotations, Reports, and Settings.
- Add isolated Coach app styles and JavaScript behavior.
- Add a Coach-specific web app manifest and service worker.
- Document the boundary between the existing scoreboard and the new Coach app.
- Do not connect Firebase yet.
- Do not modify existing scoreboard files.

## Files Changed

- `ScoreFlow-Coach/index.html`
- `ScoreFlow-Coach/style.css`
- `ScoreFlow-Coach/app.js`
- `ScoreFlow-Coach/manifest.json`
- `ScoreFlow-Coach/service-worker.js`
- `ScoreFlow-Coach/README.md`
- `docs/PRs/PR-002.md`

## Implementation Notes

The Coach app is a standalone static application inside its own folder. Navigation currently swaps between the Home dashboard and clearly labeled placeholder views. The layout includes a desktop sidebar and a mobile slide-out menu.

No Firebase configuration, authentication, team data, player data, match data, or scoreboard integration is included in this PR. Those concerns will be added through later focused pull requests.

The existing ScoreFlow scoreboard files in the repository root are intentionally untouched.
