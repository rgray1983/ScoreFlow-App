# PR-004 — Organizations, Teams, and Seasons Foundation

## Status

Ready for local testing and review.

## Branch

`agent/pr-004-organizations-teams-seasons`

## Purpose

Create the first working ScoreFlow Coach data foundation using the documented hierarchy:

```text
Organization
└── Team
    └── Season
```

This active context will later control rosters, schedules, live matches, rotations, statistics, reports, and team portal data.

## Scope

- Add strict TypeScript models for organizations, teams, seasons, and workspace state.
- Add a local-first workspace provider backed by browser `localStorage`.
- Seed one representative school, varsity team, and active season for testing.
- Build a working Teams route for creating and selecting organizations, teams, and seasons.
- Support school, club, and independent organization types.
- Store team level, abbreviation, colors, home court, and organization ownership.
- Store season name, date range, and active/upcoming/completed status.
- Keep active organization, team, and season selections after refresh.
- Update the Coach Hub from the selected team and season rather than hard-coded team copy.
- Preserve the viewport-safe mobile/iPad shell from PR-003.
- Keep the existing root scoreboard untouched.

## Local-First Behavior

All PR-004 records save to the browser under `scoreflow-coach-workspace-v1`. This is intentionally a local foundation rather than the final sync engine. Future Firebase integration should preserve these entity shapes and migrate local records into authenticated cloud collections without changing the user workflow.

## Intentionally Not Included

- Firebase synchronization
- Authentication or staff permissions
- Editing existing records
- Destructive deletion
- Full archive-management interface
- Logo file upload
- Scoreboard defaults editor
- Roster, schedule, match, rotation, or report records
